AgentGem Deployment

Cloudflare edge · Render (Oregon) · Neon Postgres · GitHub registry — from render.yaml, Dockerfile & wrangler.toml

Cloudflare (edge)

  • • Worker agentgem-web serves agentgem.ai from static assets
  • • Edge-renders /share/:id OG cards via resvg-wasm
  • • Proxies *.agentgem.ai to Render, terminates TLS
  • • Injects cf-connecting-ip → keys API rate limits

Render (Oregon)

  • • agentgem — Docker API (node:24-slim), /healthz
  • • agentgem-app — Vite/React SPA, CDN static, no sleep
  • • Free web plan sleeps after 15m idle (30–50s cold start)
  • • SERVE_CONSOLE=false: console UI stays local

State & external

  • • Neon Postgres via DATABASE_URL — durable moat data
  • • (Render free Postgres self-deletes at 30 days → Neon)
  • • GitHub agentgem-registry — public gem catalog
  • • Desktop console is local, talks to api.agentgem.ai
