AgentGem Transfer — Ticket + NATS Object Store

Store-and-forward (Option B): peers never need to be online together · broker holds ciphertext only · zero deploy for end users

Why store-and-forward

  • • Peers never online together — drop now, grab later
  • • Fits cross-device & friend/coworker equally
  • • JetStream Object Store: put / get / del, chunked
  • • Managed Synadia = zero deploy for end users

Security model

  • • Broker holds ciphertext only (AES-256-GCM)
  • • Ticket is the capability — key in URL #fragment
  • • Burn-after-fetch (del) + bucket TTL on tickets
  • • 3 layers: GCM tag · gem.lock SHAs · ed25519 sig

Build footprint

  • • New dep: NATS client only (no PAKE, no @noble/*)
  • • Crypto = Node built-in (single-use key)
  • • src/transfer/: objectStore · seal · ticket · index
  • • InMemoryObjectStore → hermetic CI; NATS test gated
