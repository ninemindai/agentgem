# Undeploy (all four backends) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax.

**Goal:** Add an "Undeploy" action to each Deploy-tab backend that removes the live cloud resource it created (eve→Vercel project, flue→Cloudflare Worker, Claude Managed agent, AgentCore harness), and fix eve so each gem deploys to its own per-gem Vercel project.

**Architecture:** Persist a small per-workspace deploy record at deploy time (resource ids/urls/names per backend). Each backend gains an `undeploy*` function that reads that record and runs the matching removal (CLI for eve/flue, API for managed/agentcore). A single `POST /api/undeploy` dispatches by target; the UI shows an Undeploy button (with confirm) when a deploy record exists.

**Tech Stack:** TypeScript (NodeNext ESM, `.js` specifiers), Zod, `@agentback/rest`, vitest + supertest, vanilla JS in `src/public/index.html`. CLIs: pinned `vercel`, project-local `wrangler`. SDKs: `@anthropic-ai/sdk` (`client.beta.agents/environments/skills`), `@aws-sdk/client-bedrock-agentcore-control`.

## Global Constraints

- ESM imports use `.js` specifiers even for `.ts` sources (NodeNext).
- Tests: `npm test` (`tsc -b && vitest run` from compiled `dist/`); scoped `npm test -- <file>`; on stale-build error `rm -rf dist tsconfig.tsbuildinfo && npm test`.
- Undeploy is DESTRUCTIVE — the UI must confirm before calling it; backends must fail safe (clear error) when no deploy record exists.
- Deploy records live under the workspace: `<workspaceDir>/.deploy/<backend>.json` (gitignored territory; workspace store is `~/.agentgem/workspaces/<name>`). `workspaceDir(name)` is exported from `src/gem/workspaces.js`.
- Reuse existing process-runner injection (`ProcessRunner`/`runToEnd`) for CLI undeploys so they're unit-testable with the fake runner in `src/gem/__tests__/run.test.ts`.
- Reuse the existing `PublishClient` (`src/publish.ts`) + the agentcore control client (`src/gem/agentcorePublish.ts`) — extend with delete methods; do not create new SDK wiring patterns.
- Vercel scope: reuse the single-team auto-detect (`parseSingleTeamScope`) + `VERCEL_SCOPE` already in `run.ts`.
- Credentials gate the same as deploy: eve→`VERCEL_TOKEN`, flue→`CLOUDFLARE_API_TOKEN`, managed→`ANTHROPIC_API_KEY`, agentcore→AWS creds.

```typescript
// src/gem/deployRecord.ts
import { existsSync, mkdirSync, readFileSync, writeFileSync, rmSync } from "node:fs";
import { dirname, join } from "node:path";
import { workspaceDir } from "./workspaces.js";

export type DeployBackend = "eve" | "flue" | "claude-managed" | "agentcore";
export interface DeployRecord {
  backend: DeployBackend; at?: string; url?: string; project?: string; worker?: string;
  agentId?: string; environmentId?: string; skillIds?: string[]; harnessId?: string;
}
function recPath(name: string, backend: DeployBackend): string {
  return join(workspaceDir(name), ".deploy", `${backend}.json`);
}
export function writeDeployRecord(name: string, rec: DeployRecord): void {
  const abs = recPath(name, rec.backend);
  mkdirSync(dirname(abs), { recursive: true });
  writeFileSync(abs, JSON.stringify(rec, null, 2) + "\n", "utf8");
}
export function readDeployRecord(name: string, backend: DeployBackend): DeployRecord | null {
  const abs = recPath(name, backend);
  try { return existsSync(abs) ? JSON.parse(readFileSync(abs, "utf8")) as DeployRecord : null; }
  catch { return null; }
}
export function clearDeployRecord(name: string, backend: DeployBackend): void {
  rmSync(recPath(name, backend), { force: true });
}
```

```typescript
export async function undeployVercel(name: string, runner: ProcessRunner = realRunner): Promise<{ removed: boolean; logTail: string[] }> {
  const token = process.env.VERCEL_TOKEN;
  if (!token) throw new Error("VERCEL_TOKEN is not set on the server — cannot undeploy from Vercel.");
  const rec = readDeployRecord(name, "eve");
  if (!rec?.project) throw new Error(`no recorded eve/Vercel deploy for '${name}'`);
  const logTail: string[] = [];
  const scope = process.env.VERCEL_SCOPE;
  const run = (s?: string) => new Promise<{ code: number; lines: string[] }>((resolve) => {
    const lines: string[] = [];
    const args = ["remove", rec.project!, "--yes", `--token=${token}`, ...(s ? [`--scope=${s}`] : [])];
    const h = runner.spawn(VERCEL_BIN, args, { cwd: workspaceDir(name), env: process.env });
    h.onLine((l) => { pushLog(logTail, l); lines.push(l); });
    h.onExit((c) => resolve({ code: c ?? 0, lines }));
  });
  let { code, lines } = await run(scope);
  if (code !== 0 && !scope) { const team = parseSingleTeamScope(lines); if (team) ({ code } = await run(team)); }
  if (code !== 0) return { removed: false, logTail };
  clearDeployRecord(name, "eve");
  return { removed: true, logTail };
}
```

```typescript
export async function undeployCloudflare(name: string, runner: ProcessRunner = realRunner): Promise<{ removed: boolean; logTail: string[] }> {
  const token = process.env.CLOUDFLARE_API_TOKEN;
  if (!token) throw new Error("CLOUDFLARE_API_TOKEN is not set on the server — cannot undeploy from Cloudflare.");
  const rec = readDeployRecord(name, "flue");
  if (!rec?.worker) throw new Error(`no recorded flue/Cloudflare deploy for '${name}'`);
  const runDir = join(workspaceDir(name), ".run", "flue");
  const logTail: string[] = [];
  const env = { ...process.env, CLOUDFLARE_API_TOKEN: token };
  const code = await new Promise<number>((resolve) => {
    const h = runner.spawn(binIn(runDir, "wrangler"), ["delete", "--name", rec.worker!, "--force"], { cwd: runDir, env });
    h.onLine((l) => pushLog(logTail, l));
    h.onExit((c) => resolve(c ?? 0));
  });
  if (code !== 0) return { removed: false, logTail };
  clearDeployRecord(name, "flue");
  return { removed: true, logTail };
}
```
